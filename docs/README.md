# Docs

This directory is the working home for fork-specific documentation and planning.

## Structure

- `planning/` - design notes, API sketches, and implementation plans
- `decisions/` - short decision records and rationale
- `references/` - external or upstream reference material summaries

## Guidance

- Keep implementation notes here rather than scattering them across ad hoc files.
- Prefer small, focused documents over one large catch-all note.
- When a document reflects an active constraint for the fork, link it from here.
